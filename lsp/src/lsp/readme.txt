Nesse exemplo clássico, é definido o L (Liskov Substitution Principle) que exemplifica o exemplo do retângulo que pode ser triângulo. No que segue pela explicação "Sempre que uma classe cliente esperar uma instância de uma classe base X, uma instância de uma subclasse Y de X deve poder ser usada no seu lugar". Nesse exemplo, porém, as classes, mesmo que com herança, relacionam-se entre sí corretamente.
